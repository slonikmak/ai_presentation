# Доклад-дополнение: Дневник одной агентской сессии (ES-3772)

**Статус:** дополнительная презентация к основному докладу «AI-агенты как ежедневный инструмент команды». Ориентирована на dev-команду.
**Источник:** реальная агентская сессия по задаче ES-3772 (PDF ToC/ToF/ToT spacing), собранная агентом в дневник `docs/diary/agent-session-diary.md` + артефакты сессии в `docs/diary/` (task, spec, plan, testing-report, export-debug доки, скриншот регрессии).
**Акцент:** не «агент сделал фичу», а демонстрация использования тулов и скиллов, петель проверки и саморефлексии агента с последующим превращением рефлексий в скиллы/инструкции. Рамка — улучшение процесса «человек + агент».
**Преза:** `presentation/diary.html`, та же дизайн-система и каркас, что у основной и SDD.

## Формат: сплит-слайды с живыми артефактами

Преза онлайновая, каждый открывает у себя. Ключевая механика:

- **Сплит-раскладка**: слева тезисы слайда, справа скролл-панель с реальным артефактом сессии (markdown отрендерен внутри презы). Зритель сам скроллит документ, не зависит от спикера.
- Артефакты зашиты в HTML инлайн (single-file, без билда); лёгкий markdown-рендер (заголовки, код-блоки, списки, цитаты).
- Hash-навигация (`diary.html#12`) — можно кидать ссылку на конкретный слайд.
- Клик/скролл внутри панели артефакта не листает слайды.
- Скриншот регрессии `img.png` копируется в `presentation/assets/diary-regression.png`.

## Тезисы-акценты (для подачи)

- **Кульминация** — пара слайдов 15–16: «все тесты зелёные, все файлы правильные, PDF плохой». Момент, где видно различие «код выглядит правильным» vs «фактический HTML в рендерере».
- **Три регрессии — три уровня совместимости**: CSS-свойства → поведение рендерера → template placeholders. Ни одна не была видна из unit-тестов.
- **Человек в сессии** — не наблюдатель, а сенсор (визуальная проверка PDF) и источник product-рисков; его цитаты разворачивали сессию.
- **Скиллы** — это markdown с процессом команды; сессия их использует и (через рефлексию) производит новые.
- **Внешняя рефлексия** — дешёвый шаг: второй агент по готовым артефактам находит системное, чего не видно изнутри сессии.
