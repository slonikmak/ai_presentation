# CLAUDE.md

Проект — презентация (доклад) **«AI-агенты как ежедневный инструмент команды»**. Язык проекта — русский; общайся и пиши контент по-русски.

## Главное правило: меняй только по прямой просьбе

Ничего не редактируй по своей инициативе. Меняй файлы **только** когда пользователь явно просит внести изменение («добавь», «убери», «перепиши», «исправь»). Вопросы, обсуждение, поиск — без правок файлов.

## Поток изменений: доки → преза

Источник истины — `docs/`. Презентация — производная от доков. Любое содержательное изменение идёт строго в два шага:

1. **Сначала доки.** Внеси изменение в оба документа в `docs/` (см. ниже), сохраняя их согласованность между собой.
2. **Потом преза.** Перенеси то же изменение в `presentation/index.html`.

Не правь презентацию, минуя доки. Если правка касается только верстки/дизайна (не смысла) — это исключение, доки трогать не нужно.

## Контекст компании (важно для содержания)

Маленькая компания, узкий рынок, несколько клиентов; продукты для узкого круга пользователей. Все примеры и кейсы в докладе строить под эту реальность: данные и выгрузки по конкретному клиенту, разбор инцидентов, использование фич, документация, коммуникации под конкретный аккаунт. **Нерелевантны** примеры из массовых продуктов: пользовательские сегменты, конверсии, воронки, «сколько пользователей в стейте N» и т.п.

## Презентация (`presentation/index.html`)

- Один HTML-файл: 45 слайдов, инлайновые CSS-токены и JS-навигация. Никакого билда — открывается в браузере напрямую.
- Слайды размечены комментариями-разделителями `<!-- ===== N · НАЗВАНИЕ ===== -->`; каждый слайд — `<section class="slide">` (модификаторы `dark`, `soft`). Активный слайд — класс `active`.
- Счётчик в HUD (`NN / 45`) и точки навигации генерятся из числа `.slide` автоматически — при добавлении/удалении слайда ничего вручную пересчитывать не нужно.
- Навигация: ←/→, Space, Home/End, F — фуллскрин, клик по половинам экрана.

## Дизайн-система

Бренд **Vibe Coding** уже зашит в токены `:root` внутри `index.html` — используй существующие переменные, не вводи новые цвета/шрифты:

- Холст белый / `#F4F5F7`; акцент малиново-маджента `#E6006B` (`--accent`); нейтрали-сланцы `#2B3340`/`#3D4757`.
- Острые углы (radius 2–4px), мотив слэша `/`, без эмодзи.
- Шрифты: в бренде — Hanken Grotesk + JetBrains Mono, но он **не покрывает кириллицу**, поэтому в презе подставлены **Manrope** (текст) + **JetBrains Mono** (код). Для кириллицы оставайся на Manrope.
- Тон: прямой, инженер-инженеру; sentence-case заголовки, UPPERCASE-надзаголовки (`.kicker`) и кнопки.

Полная дизайн-система — в скилле `.claude/skills/Vibe Coding Design System/` (`README.md`, `colors_and_type.css`, `ui_kits/`, `preview/`). Сверяйся с ней при новых визуальных блоках. Иконки — Lucide, обводка 1.75–2px, `currentColor`.
